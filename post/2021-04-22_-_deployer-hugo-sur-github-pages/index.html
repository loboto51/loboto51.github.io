Déployer un site web statique Hugo sur github Pages

2021-04-22 · 2 min read  · hugo · github pages ·
Partager sur:

Préparer dépôt github :

Pour être disponible via Github Pages le repository github doit être public et créé avec le nom :

{user github}.github.io

Créer une copie locale :

1mkdir loboto51.github.io
2cd loboto51.github.io
3echo "# loboto51.github.io" >> README.md
4git init
5git add README.md
6git commit -m "first commit"
7git remote add origin https://github.com/loboto51/loboto51.github.io.git
8git branch -M main
9git push -u origin main

Hugo :

Package :

Récupérer dernière version (ici ubuntu) :

1wget https://github.com/gohugoio/hugo/releases/download/v0.82.1/hugo_extended_0.82.1_Linux-64bit.deb
2sudo dpkg -i hugo_extended_0.82.1_Linux-64bit.deb

si erreurs de dépendances :

1sudo apt-get install -f

Créer un socle de site :

1cd loboto51.github.io
2hugo new site .

Installer un thème (ici hugo-clarity):

1cd loboto51.github.io/themes
2git clone https://github.com/chipzoller/hugo-clarity.git

Enlever les éléments inutiles du theme :

1rm -Rf hugo-clarity/.git*
2rm -Rf hugo-clarity/exampleSite

Pour créer une page de test (sinon n'importe que .md dans le rep post/ fait l'affaire) :

1hugo new post/testpage.md

Vérification que ça marche :

Lancer serveur Hugo en local pour tester :

1hugo server -D

ouvrir : http://localhost:1313/

Si tout se passe bien on peut poursuivre.

Génération automatique des pages sur le dépôt :

Ajouter un fichier de conf au dépôt pour générer automatiquement les pages Hugo à chaque push (doc Hugo : hosting on github) :

1cd loboto51.github.io
2mkdir -p .github/workflows/
3touch .github/workflows/gh-pages.yml

Y mettre la conf auto suivante. Elle génère les pages statiques Hugo dans une branche gh-pages :

 1name: github pages
 2
 3on:
 4  push:
 5    branches:
 6      - main  # Set a branch to deploy
 7
 8jobs:
 9  deploy:
10    runs-on: ubuntu-18.04
11    steps:
12      - uses: actions/checkout@v2
13        with:
14          submodules: true  # Fetch Hugo themes (true OR recursive)
15          fetch-depth: 0    # Fetch all history for .GitInfo and .Lastmod
16
17      - name: Setup Hugo
18        uses: peaceiris/actions-hugo@v2
19        with:
20          hugo-version: 'latest'
21          # extended: true
22
23      - name: Build
24        run: hugo --minify
25
26      - name: Deploy
27        uses: peaceiris/actions-gh-pages@v3
28        with:
29          github_token: ${{ secrets.GITHUB_TOKEN }}
30          publish_dir: ./public

Push et dernières confs sur github :

1cd loboto51.github.io
2git commit -a -m "init"
3git push

Se connecter au dépôt git, dans "Action" on voit le traitement se dérouler (et éventuellement planter). Aller dans la conf du dépot : Pages > Source Choisir :

[Branch : gh-pages] [/root]

Aller sur le site : loboto51.github.io En théorie tout marche.
